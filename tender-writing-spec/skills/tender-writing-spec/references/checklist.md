# Tender Writing Checklist

Use this checklist before delivering tender text or modifying a bid document.

## Section Control

- Confirm the exact chapter, heading level, and insertion point.
- Confirm whether the task is `新增`, `局部修改`, or `整段重写`.
- Keep surrounding numbering, outline hierarchy, and chapter order unchanged unless the user explicitly asks otherwise.

## Content Consistency

- Confirm the latest final quotation or procurement list before writing.
- Confirm whether the project scope includes hardware, software only, cloud services, API services, or integration only.
- Confirm whether there is an existing Agent platform, knowledge base, or business system that must be reused.
- Confirm the project time span, such as `建设1年 + 运维3年`, and keep all service descriptions aligned.

## Writing Quality

- Make every paragraph answer one practical question: what is built, how it works, what it connects to, and what value it provides.
- Prefer implementable phrasing over promotional language.
- Keep smart-city/IOC wording grounded in operations, command, dispatch, safety, maintenance, data, and event handling.
- Avoid inventing unsupported performance indicators or product capabilities.

## AI Section Check

- Clarify user entry point: voice, text, dashboard, mobile, backend.
- Clarify recognition or reasoning mechanism.
- Clarify connected data source or subsystem.
- Clarify feedback or execution result.
- Clarify permissions, confirmation, and audit controls.

## DOCX Editing Check

- Back up the original file before major edits.
- Preserve font, font size, paragraph style, and layout.
- Ensure new paragraphs did not accidentally inherit a heading style.
- Ensure no original text was deleted by mistake.
- Re-read the edited region after saving.
