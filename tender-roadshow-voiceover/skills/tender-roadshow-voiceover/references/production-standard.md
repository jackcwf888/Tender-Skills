# Production Standard

## Audience

Use this profile for:
- tender demos
- bid defense videos
- project capability roadshows
- expert review presentations
- client walkthrough videos with narration

## Listening Goal

The listener should hear:
- stable project-presenter confidence
- clear and complete pronunciation
- enough pause to follow screen transitions
- technical seriousness without sounding stiff

The listener should not hear:
- promotional hype
- radio-ad energy
- rushed phrases
- flat robotic sentence endings

## Voice Profile

- Default voice: `zh-CN-YunyangNeural`
- Gender preference: male unless the user asks otherwise
- Speed: standard by default
- Pitch: neutral to slightly lowered
- Volume: neutral

## Pause Standard

- Keep a visible pause between narration rows.
- Start from about `650ms` between rows for standard roadshow use.
- Shorten only when strict duration limits require it.
- Lengthen when the visual transitions are slow or the script is especially dense.

## Script Hygiene

- Use one roadshow beat per row when possible.
- Replace internal hard line breaks with spoken commas unless the break is intended as a dramatic stop.
- Preserve technical terms, model levels, building names, and module names exactly.
- Avoid rewriting user-provided wording unless the user asks for copy polishing.

## Duration Tradeoff

When the user gives a target duration:
- first adjust pauses
- second apply moderate speaking-rate changes
- only after that consider stronger acceleration

For expert-facing tender material, intelligibility outranks exact timing unless the timing is a hard production constraint.
