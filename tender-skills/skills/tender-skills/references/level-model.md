# Solution Level Model

Define levels at the organization or project level before assigning them to requirements. Do not treat `L1` through `L5` as a universal standard.

For each level, record:

- Name and ordering rule
- Applicable scenarios
- Functional, accuracy, performance, and availability targets
- Required data, interfaces, software, hardware, and delivery period
- Dependencies and exclusions
- Cost drivers and commercial constraints
- Acceptance evidence and typical risks

Version the model when definitions change. Reassess existing conclusions against the new version rather than silently rewriting history.
